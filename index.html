Static blog demo for GitHub Pages

A cozy one-page blog built with plain HTML, Tailwind CDN, and a clean dark mode.

This is a dead-simple starter you can throw into a GitHub repository and publish with GitHub Pages. No build step. No framework. Just one HTML file with a bloggy vibe.

Journal April 24, 2026 • 5 min read

Today I learned that the internet still has room for tiny, cozy websites.

My day started the way many excellent digital days do: with a simple task that could have become an overcomplicated one. The request was humble — make an example GitHub Page, give it a blog feel, keep it to plain HTML, and if possible, use the Tailwind CDN. Beautiful. No bundlers. No package manager existentialism. No twelve-step deployment ritual.

There is something deeply charming about a static page. You write a file, push it to a repository, flip the GitHub Pages switch, and suddenly your words are standing on the open web wearing a clean little outfit. It feels almost suspiciously straightforward now.

Sometimes the nicest websites are just a good font stack, generous spacing, and a person with something to say.

So I leaned into the vibe: soft colors, rounded corners, roomy paragraphs, and a layout that feels a bit like a notebook left open on a kitchen table. The kind of page where you might expect to read thoughts about weather, dogs, code, toast, and whether the modern web really needed this many pop-ups.

I also spent a moment appreciating the fact that GitHub Pages still encourages small publishing. Not everything needs a backend, analytics pipeline, and venture capital. Sometimes you just want an internet corner. A little place. A webpage with opinions and maybe one nice photograph of a dog.

Anyway, that was my day: making a tiny thing on purpose, and enjoying how light it felt. If this page goes live, I hope it inspires at least one more lovely, unnecessary, handmade website.

Notes

  • • Tailwind is loaded from the CDN, so there is no setup step.
  • • Everything lives in a single index.html file.
  • • A theme toggle stores your choice in localStorage.

How to publish

  1. 1. Create a GitHub repository.
  2. 2. Upload this index.html to the repo root.
  3. 3. Open Settings → Pages.
  4. 4. Under Build and deployment, choose Deploy from a branch.
  5. 5. Select your branch, usually main, and folder /root.
  6. 6. Save, then wait a minute for the site to go live.
